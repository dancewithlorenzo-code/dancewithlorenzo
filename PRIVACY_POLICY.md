# Privacy Policy for Dance with Lorenzo Tokyo

**Last Updated:** March 2, 2026

## Introduction

Dance with Lorenzo Tokyo ("we," "our," or "us") operates the Dance with Lorenzo Tokyo mobile application (the "App"). This Privacy Policy explains how we collect, use, disclose, and safeguard your information when you use our App.

Please read this privacy policy carefully. If you do not agree with the terms of this privacy policy, please do not access the App.

## Information We Collect

### Personal Information You Provide

When you register for and use our App, we collect the following personal information:

- **Account Information:** Email address, username, password (encrypted)
- **Profile Information:** Display name, language preference
- **Payment Information:** Processed securely through Stripe (see Payment Processing section)
- **Booking Information:** Class bookings, private lesson requests, attendance records
- **Communication Preferences:** Push notification settings and device tokens

### Automatically Collected Information

When you use the App, we automatically collect:

- **Device Information:** Device type, operating system, unique device identifiers
- **Usage Data:** App features accessed, classes viewed, booking patterns
- **Location Data:** Class location information you select (not real-time tracking)
- **QR Code Scanning:** Check-in timestamps and class attendance verification

## How We Use Your Information

We use the collected information for the following purposes:

### Service Delivery
- Create and manage your account
- Process class bookings and token purchases
- Verify check-ins via QR code scanning
- Send booking confirmations and class reminders
- Process refunds and cancellations

### Communication
- Send push notifications about upcoming classes (24 hours before)
- Notify you about booking confirmations
- Update you on private lesson request status
- Send payment receipts via email
- Provide customer support

### Analytics and Improvement
- Generate personal analytics (attendance rate, favorite classes)
- Improve our services and user experience
- Analyze booking patterns and class popularity
- Develop new features

### Legal and Security
- Comply with legal obligations
- Prevent fraud and unauthorized access
- Protect user safety and App security
- Enforce our Terms of Service

## Payment Processing

We use **Stripe** as our third-party payment processor. When you make a payment:

- **Payment information** (credit card details) is collected and processed directly by Stripe
- We **never store** your full credit card information on our servers
- We receive only transaction confirmations and payment metadata from Stripe
- Stripe's processing is subject to their Privacy Policy: https://stripe.com/privacy

### Payment Types We Process
- Token package purchases (¥33,000 for 4 BMD tokens)
- Workshop bookings (¥15,000 or ¥12,000 depending on participant count)
- Private lesson payments (¥40,000 per lesson)

## Push Notifications

We send push notifications for:

- **Class Reminders:** 24 hours before booked classes
- **Booking Confirmations:** Immediately after successful bookings
- **Private Lesson Updates:** When Lorenzo approves or rejects requests
- **Payment Confirmations:** After successful transactions

You can manage notification preferences in your device settings. Disabling notifications may affect your ability to receive important class updates.

### Technical Implementation
- We use **Expo Push Notification Service** to deliver notifications
- Your device push token is stored securely in our database
- Notifications are sent only for events relevant to your bookings

## Data Storage and Security

### Where We Store Data

Your data is stored on **OnSpace Cloud** (Supabase infrastructure):
- Database servers located in secure, certified data centers
- Data encrypted in transit using SSL/TLS
- Data encrypted at rest using industry-standard encryption

### Security Measures

We implement appropriate security measures including:
- Password encryption using bcrypt hashing
- Row Level Security (RLS) policies on all database tables
- Secure authentication using email OTP verification
- Regular security audits and updates
- Access controls and user authentication

### Data Retention

- **Active Accounts:** Data retained while your account is active
- **Deleted Accounts:** Personal data deleted within 30 days of account deletion
- **Transaction Records:** Payment records retained for 7 years for tax/legal compliance
- **Attendance Records:** Retained for service quality and analytics purposes
