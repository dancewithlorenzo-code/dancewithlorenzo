## 🔐 Production APK Signing (CRITICAL)

### Step 1: Generate Signing Keystore

Run this command in your project root:

```bash
# Generate production keystore (one-time setup)
keytool -genkeypair -v -storetype PKCS12 \
  -keystore dance-lorenzo-release.keystore \
  -alias dance-lorenzo-key \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -storepass YOUR_STRONG_PASSWORD \
  -keypass YOUR_STRONG_PASSWORD \
  -dname "CN=Lorenzo Schmidt, OU=Dance with Lorenzo, O=Dance with Lorenzo, L=Tokyo, ST=Tokyo, C=JP"
```

**⚠️ IMPORTANT**:
- Replace `YOUR_STRONG_PASSWORD` with a **strong, unique password**
- **SAVE THIS PASSWORD SECURELY** - you cannot recover it if lost
- Store `dance-lorenzo-release.keystore` in a **secure location**
- **NEVER commit keystore to Git** (already in .gitignore)
- **Backup keystore file** - losing it means you can't update your app!

### Step 2: Configure Gradle for Signing

Create `android/gradle.properties` (if not exists):

```properties
# Signing Config
DANCE_LORENZO_RELEASE_STORE_FILE=../dance-lorenzo-release.keystore
DANCE_LORENZO_RELEASE_KEY_ALIAS=dance-lorenzo-key
DANCE_LORENZO_RELEASE_STORE_PASSWORD=YOUR_STRONG_PASSWORD
DANCE_LORENZO_RELEASE_KEY_PASSWORD=YOUR_STRONG_PASSWORD
```

**⚠️ SECURITY NOTE**: Keep `gradle.properties` LOCAL ONLY - do not commit!

### Step 3: Build Production APK

```bash
# Navigate to project root
cd /path/to/dance-with-lorenzo

# Install dependencies
npm install

# Build production APK
npx expo run:android --variant release
```

**Alternative - EAS Build (Recommended)**:

```bash
# Install EAS CLI
npm install -g eas-cli

# Login to Expo
eas login

# Configure EAS
eas build:configure

# Build production APK
eas build --platform android --profile production
```

**Output location**:
- Local build: `android/app/build/outputs/apk/release/app-release.apk`
- EAS build: Download from build dashboard

### Step 4: Test APK Before Submission

```bash
# Install on test device
adb install -r android/app/build/outputs/apk/release/app-release.apk

# Or transfer APK to device and install manually
```

**Testing checklist**:
- ✅ App launches without crashes
- ✅ All features work (login, booking, payment)
- ✅ Push notifications work
- ✅ QR scanner functions properly
- ✅ Stripe payments process correctly
- ✅ No debug/test data visible
